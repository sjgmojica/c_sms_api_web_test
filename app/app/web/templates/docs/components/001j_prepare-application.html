How To Prepare Your Application

This is how your application will communicate with the Chikka SMS API:

Receiving and Replying to an SMS

  1. User sends an SMS to your access code (Chikka SMS API).
  2. Your Message Receiver component receives the SMS from the Chikka SMS API.
  3. Your Message Receiver component replies to the SMS and sends it to the Chikka SMS API.
  4. The Chikka SMS API delivers your reply message to the User.
  5. Your Notification Receiver component receives a delivery notification from the Chikka SMS API.

Sending an SMS

  1. Message Sender sends an SMS to the Chikka SMS API.
  2. The Chikka SMS API delivers your message to the User.
  3. Your Notification Receiver component receives a delivery notification from the Chikka SMS API.

Your application needs to have the following components, to be able to use the Chikka SMS API:

  1. Message Receiver - a web application designed to accept and receive HTTP POST requests (messages from users) FROM the Chikka SMS API and reply to messages via HTTP POST requests.

    The Chikka SMS API is asynchronous; it is expected that your Message Receiver component must be able to handle duplicate messages. In order to detect duplicates, refer to the request_id parameter; this is always unique per incoming message.

  2. Message Sender - a web application designed to send HTTP POST requests (messages from YOU) TO the Chikka SMS API.

  3. Notification Receiver - a web application designed to accept and receive HTTP POST requests (delivery notifications) FROM the Chikka SMS API.

    Here's a sample code for PHP:

    • PHP
    • Python
    • Java

    The Chikka SMS API is asynchronous; it is expected that your Notification Receiver component must be able to handle duplicate notifications. In order to detect duplicates, refer to the message_id parameter; this is the unique ID provided by the API user to track outgoing messages.

After preparing your application, you need to submit the corresponding URLS of the following components: Message Receiver and Notification Receiver to the API Settings in your dashboard.
